What Is This Module

Complete visibility into
every system event.

ZarklyX System Monitoring captures two distinct layers of data — internal software operations and real human behavior — forming a complete, auditable record of everything that happens inside your platform.

System Logs track every internal operation performed within the software. Every time someone interacts with the data, a log is automatically created and timestamped. Whether new information is being added, existing records are being modified, or data is being removed, every data-level change is captured with full traceability and linked to the user who triggered it.

The Activity Logger works at a different layer — it monitors how a person actually uses the interface from the moment they log in until they log out. Rather than tracking data changes, it captures behavioral signals: where users move their cursor, what they type, which elements they click, how long they hover over a component, and everything else they do while active in the system.

🗂️ System Logs

Tracks Create, Update, and Delete operations — every time data is added, changed, or removed, a timestamped log entry is written automatically.

🖱️ Activity Logger

Monitors user behavior throughout every session — login and logout times, mouse movements, key presses, hover events, clicks, and general activity.

🔍 Full Audit Trail

Every log and activity event is stored, queryable, and linked to the responsible user, providing a complete audit trail that is always accessible.

Step-by-Step Process

How the System Works

From the moment a user logs in to the moment they close their session, every operation and interaction flows through two parallel tracking systems that run silently in the background.

1

User Logs In

The Activity Logger immediately records the login event with a precise timestamp, opening the session window and beginning continuous behavioral monitoring across the interface.

2

Data Interactions Logged

Every time the user adds, modifies, or removes information, the System Logger writes a Create, Update, or Delete entry — capturing the operation, module, user, and timestamp automatically.

3

Behavior Continuously Tracked

Simultaneously, the Activity Logger records all interface interactions — mouse movements, key presses, hover durations, and click events — building a complete behavioral map of the session.

4

Session Ends & Logs Finalized

When the user logs out, the logout event is recorded, closing the session record. All logs from both systems are immediately available for review, filtering, and audit in the dashboard.

System Logs

Every data change,
permanently recorded.

System Logs track every internal operation performed within the software. Every time someone interacts with the data, a log is created. Each log entry captures the operation type, the user responsible, the affected module, and the precise time it occurred — giving you a complete, immutable record of all data-level activity across the platform.

Create

A Create log is generated every time new information is added to the system. Whether a user is adding a new record, registering a new entry, or writing a new data object, the system captures this operation instantly. The log records who performed the action, which module was affected, and the exact timestamp at which the new information was written — ensuring that every addition to the data layer is fully traceable and accountable.

Update

An Update log is recorded each time existing information within the system is changed. Any modification made to a previously stored record — regardless of how minor the change — triggers an Update entry in the System Log. This ensures a complete trail of alterations over time, allowing administrators to see not just what data exists now, but how it has evolved and which users were responsible for each change that was made.

Delete

A Delete log is captured whenever information is permanently removed from the system. This log records the deletion event precisely, documenting what was removed, who removed it, from which module, and when the removal occurred. No data can be deleted silently — every Delete operation creates a clear and permanent audit entry that remains visible in the System Log even after the original data is no longer present in the system.

Activity Logger

User behavior, fully visible.
From login to logout.

The Activity Logger tracks user behavior from the moment they log in. It monitors how a person actually uses the interface — capturing a continuous, real-time record of everything a user does while active in the system, far beyond just what data they changed.

Login / Logout

The Activity Logger records the precise timestamps of when a user starts and ends their session. The Login event is captured the moment a user authenticates and enters the system, while the Logout event is recorded when they formally end their session. Together, these two events establish the exact active window of every user session, providing a clear and auditable record of when each user was present in the platform.

Mouse Movements

Mouse Movement tracking captures where the user moves their cursor on the screen throughout their session. This continuous tracking reveals navigation patterns, the areas of the interface that receive the most attention, and how users physically move through the platform's layout. Mouse movement data provides a spatial understanding of user behavior that complements the discrete event logs recorded for clicks, hovers, and keyboard interactions.

Key Presses

The Activity Logger records any typing or keyboard activity performed by the user during their session. Every key press — whether it is text being entered into a form field, a search query being typed, or a keyboard shortcut being used — is logged as a Key Press event. This captures the full scope of keyboard-driven interactions, offering visibility into how users engage with input-heavy workflows and which areas of the system they interact with through typing.

Hover

Hover events are captured when a user holds their cursor over a specific element for a particular time or duration. Unlike a brief pass of the cursor, a Hover event represents sustained attention — the user's cursor resting over a button, card, tooltip, or interface element long enough to indicate intent or interest. Recording hover events reveals which parts of the interface attract attention even before a click occurs, providing a deeper layer of behavioral context beyond explicit interactions.

Click

A Click log is formed every time the user clicks on any element within the interface. Each individual click interaction is recorded as its own event entry, capturing which element was clicked, by which user, and at what point during the session. Click events represent the most direct and deliberate form of user interaction, making them a primary indicator of what actions users are actually taking and which interface elements are driving the most engagement throughout each session.

General Activity

General Activity covers the overall tracking of everything the user is doing while active in the system. Beyond the specific categorized events — clicks, key presses, hovers, and mouse movements — General Activity provides a continuous layer of presence and engagement monitoring. It ensures that the full scope of a user's active session is captured, even when the interaction does not fall neatly into one of the defined event categories, giving administrators a complete picture of user presence and behavior at all times.
